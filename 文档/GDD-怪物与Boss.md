## 2. 怪物AI详细

### 2.1 通用AI状态机

```
           ┌─────────┐
           │  IDLE   │ ← 生成后短暂停留(0.2-0.5s随机)
           └────┬────┘
                ▼
           ┌─────────┐
    ┌─────→│  CHASE  │ ← 持续追踪玩家
    │      └────┬────┘
    │           │ 距离≤攻击距离
    │           ▼
    │      ┌──────────┐
    │      │ATTACK_   │ ← 前摇 0.3s
    │      │WINDUP    │
    │      └────┬─────┘
    │           │ 前摇结束
    │           ▼
    │      ┌──────────┐
    │      │ ATTACK   │ ← 判定命中 → 闪避roll → 扣血
    │      └────┬─────┘
    │           │
    │           ▼
    │      ┌──────────┐
    │      │ COOLDOWN │ ← 后摇 0.5s
    │      └────┬─────┘
    │           │ CD结束, 距离仍≤攻击距离 → 回到WINDUP
    └───────────┘ CD结束, 距离>攻击距离 → 回到CHASE
```

### 2.2 各类型特殊AI

#### 快速怪
- 无攻击前摇（接触即伤害）
- 碰撞半径小(10px)，容易从剑之间穿过
- CHASE 状态下移速快，需要优先处理

#### 坦克怪
- 每 5 秒触发一次 **冲刺**
  - 向玩家方向直线冲刺，350px/s，持续 0.5s
  - 冲刺期间碰撞伤害 = 基础伤害 × 1.5
  - 冲刺结束有 1s 硬直（移速降至 10px/s）
- 碰撞半径大(20px)，需要更远距离躲避

#### 自爆怪
- 距离玩家 ≤ 30px 时开始自爆倒计时
- 倒计时 3 秒，期间身体脉冲发光（越来越亮）
- 自爆伤害 20，AOE 范围 60px 圆形
- **可以被击杀阻止自爆**（重要操作空间！）
- 被击杀时不触发自爆

#### 远程怪
- 距离玩家 ≤ 120px 时停步
- 每 1.5s 发射弹道（紫色光球，速度 300px/s）
- 弹道碰撞半径 8px
- 弹道可被闪避（闪避判定同近战攻击）
- 同一时间最多 3 个远程怪在射击（防止弹幕过多）
- 距离 > 120px 时恢复追踪

## 3. 波次系统

### 3.1 波次参数

```
波次    怪物总数   特殊怪比例   HP倍率   伤害倍率   速度倍率   经验倍率
────────────────────────────────────────────────────────────────────
第1波   15        0%          1.0x     1.0x       1.0x       1.0x
第2波   20        15%         1.2x     1.1x       1.05x      1.1x
第3波   25        30%         1.5x     1.2x       1.1x       1.2x
第4波   30        40%         1.8x     1.35x      1.15x      1.3x
第5波   38        50%         2.2x     1.5x       1.2x       1.4x
第6波   45        60%         2.8x     1.7x       1.25x      1.5x
```

### 3.2 波次刷新规则

```
每波开始：
  1. 在画面边缘随机位置生成怪物（距玩家 > 200px）
  2. 怪物类型分配：
     - specialCount = floor(totalCount × 特殊怪比例)
     - normalCount = totalCount - specialCount
     - 特殊怪 4 种类型均匀分配
  3. 每只怪有 0-0.5s 的随机生成延迟（防止同帧刷出）
  4. 播放波次提示动画："第 N 波！"

波次转换：
  - 当前波次怪物全部击杀 → 立即进入下一波倒计时(1s)
  - 若 40s 倒计时结束还有残留 → 下一波叠加刷新
  - 第 6 波清完 → 3s 后刷出 Boss
```

### 3.3 波次间时间线

```
0s ─────────── 40s ─────────── 80s ─────────── 120s
│              │              │               │
第1波           第2波           第3波            第4波
│              │              │               │
│ 清完即转换    │ 清完即转换    │               │
└─ 可提前进入   └─ 可提前进入   └─ ...

残存怪物 + 新波怪物可共存
```

## 4. Boss 设计

### 4.1 Boss 概览

```
Boss名称：暗影聚合体 (Shadow Amalgam)

外观：大型暗紫色不规则形体，内部有旋转的暗影核心
背景音乐变化：Boss 出场时 BGM 切换为 Boss 战专属音乐
登场方式：画面中央出现裂隙 → Boss 从中浮现(1.5s 登场动画)
```

### 4.2 Boss 数值

| 属性 | 值 | 说明 |
|------|-----|------|
| HP | 500 | 约 6 波怪物 HP 总和的 40% |
| 伤害 | 20 | 一击打掉玩家 20% 基础 HP |
| 移动速度 | 70px/s | 比玩家慢，但用技能追赶 |
| 碰撞半径 | 36px | 大体型 |
| 攻击距离 | 50px | 近战型 |
| 攻击前摇 | 0.5s | 有明显抬手动画 |
| 攻击后摇 | 1.0s | 攻击后有硬直窗口 |
| 闪避率 | 0% | Boss 无法闪避 |
| 经验掉落 | 200 | 足够升 2-3 级 |

### 4.3 Boss 三阶段行为

```
═══════════════════════════════════════════
阶段1: HP 100% → 60%（追逐阶段）
═══════════════════════════════════════════

循环执行（每轮约 5-6s）:
  追踪玩家 3s
  → 蓄力 0.8s（身体发光+变大，地面显示冲刺方向指示线）
  → 冲刺: 400px/s 持续 1s（穿过玩家时造成伤害）
  → 短暂停顿 0.5s
  → 追踪玩家 3s
  → 近战攻击（50px 范围）
  → 循环

策略提示：冲刺前有方向指示线，垂直走位可轻松躲避。

═══════════════════════════════════════════
阶段2: HP 60% → 25%（弹幕阶段）
═══════════════════════════════════════════

保留阶段1所有行为，新增:
  每 8s 释放暗影弹幕:
    - 6 方向扇形弹幕（暗影球）
    - 弹道速度 200px/s
    - 每颗伤害 8
    - 弹幕发射前 Boss 短暂静止(0.6s 前摇)
  
  冲刺冷却从 5s → 3s（冲刺更频繁）

策略提示：弹幕之间有空隙，站远一点好躲。

═══════════════════════════════════════════
阶段3: HP 25% → 0%（狂暴阶段）
═══════════════════════════════════════════

全属性增强:
  移速 +30% → 91px/s
  攻击力 +50% → 伤害 30

技能变化:
  弹幕: 6方向 → 8方向
  每 12s 召唤 4 只暗影小怪（第6波强度的普通小怪）
  冲刺冷却: 2.5s

视觉: Boss 身体变为暗红色，持续散发黑烟粒子

策略提示：先清小怪！被小怪+Boss同时夹击必死。
```

### 4.4 Boss 阶段转换

```
阶段转换时:
  - Boss 短暂无敌 0.5s
  - 播放转阶段动画（颜色变化 + 全屏震动）
  - 清除 Boss 当前所有弹幕
  - 重置技能冷却

阶段转换 HP 阈值检测:
  每帧检查 HP%，达到阈值立即转换（不等到当前技能结束）
```

## 5. 怪物生成与管理

### 5.1 安全生成区域

```
生成位置约束:
  distance(player, spawnPoint) ≥ 200px  安全距离
  spawnPoint 必须在画布内 (margin 10px)
  
生成策略:
  画面边缘随机取点 → 验证距离 ≥ 200px
  若不满足 → 重新取点 (最多尝试 50 次)
```

### 5.2 同屏怪物数量

```
最大同屏数: 80 只
超过时: 新生成的怪物加入等待队列
当前怪物死亡 → 从队列中取出补充
```

### 5.3 性能优化建议

```
碰撞检测:
  - 空间哈希: 将画布划分为 100×100px 的网格
  - 剑与怪物碰撞: 仅检测剑所在网格 ± 相邻网格
  - 怪物间不检测碰撞（降低计算量）

渲染:
  - 仅渲染在画布范围内的怪物
  - 怪物合批渲染（同类型同颜色）
  - 伤害数字使用对象池
```
